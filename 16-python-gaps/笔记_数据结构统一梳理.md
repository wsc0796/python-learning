# 数据结构统一梳理：列表 · 元组 · 字典 · 集合

---

## 核心思想：序列 vs 映射

Python 的内置数据结构分两大类：

```
序列（有序的，按位置存取）
  ├── 列表（list）  → 可变的
  ├── 元组（tuple） → 不可变的
  └── 字符串（str） → 不可变的（详见 24-string-methods）
  
映射（无序的，按键存取）
  ├── 字典（dict） → 键唯一，可变
  └── 集合（set）  → 只有键没有值，去重
```

---

## 一、共同操作（它们都支持）

| 操作 | 列表 | 元组 | 字典 | 集合 |
|------|------|------|------|------|
| `len()` | ✅ | ✅ | ✅ | ✅ |
| `in` 判断 | ✅ 查值 | ✅ 查值 | ✅ 查键 | ✅ 查值 |
| `for x in` 遍历 | ✅ | ✅ | ✅ 遍历键 | ✅ |
| `+` 拼接 | ✅ | ✅ | ❌ | ❌ |
| `*` 重复 | ✅ | ✅ | ❌ | ❌ |
| 索引 `[i]` | ✅ | ✅ | ❌ | ❌ |
| 切片 `[i:j]` | ✅ | ✅ | ❌ | ❌ |
| `.copy()` | ✅ | ❌ | ✅ | ✅ |

**关键：** 列表和元组共享索引/切片/拼接，字典和集合没有"第几个"的概念。

---

## 二、创建方式对照

| 方式 | 列表 | 元组 | 字典 | 集合 |
|------|------|------|------|------|
| 字面量 | `[1,2,3]` | `(1,2,3)` | `{"a":1}` | `{1,2,3}` |
| 空 | `[]` / `list()` | `()` / `tuple()` | `{}` / `dict()` | `set()` ❌ `{}` 是空字典 |
| 推导式 | `[x for x in r]` | `tuple(x for x in r)` | `{k:v for k,v in r}` | `{x for x in r}` |
| 互转 | `list(t)` | `tuple(l)` | `dict(zip(keys, vals))` | `set(l)` |

**注意：** `{}` 是空字典，空集合只能用 `set()`。

---

## 三、增删改查对照

### 新增

| 操作 | 列表 | 字典 | 集合 |
|------|------|------|------|
| 尾部追加 | `.append(x)` | — | `.add(x)` |
| 指定位置插入 | `.insert(i, x)` | — | — |
| 扩展多个 | `.extend([a,b])` | `.update({k:v})` | `.update({a,b})` |
| 键值对 | — | `d["key"] = value` | — |

**元组不可修改，没有这些方法。**

### 删除

| 操作 | 列表 | 字典 | 集合 |
|------|------|------|------|
| 按值删 | `.remove(x)` | — | `.remove(x)` / `.discard(x)` |
| 按索引删 | `.pop(i)` / `del l[i]` | — | — |
| 按键删 | — | `.pop(key)` / `del d[key]` | — |
| 清空 | `.clear()` | `.clear()` | `.clear()` |

### 查询

| 操作 | 列表/元组 | 字典 | 集合 |
|------|-----------|------|------|
| 按位置 | `l[i]` | ❌ | ❌ |
| 按键 | ❌ | `d["key"]` / `d.get("key")` | ❌ |
| 是否存在 | `x in l` | `"key" in d` | `x in s` |
| 查索引 | `.index(x)` | ❌ | ❌ |
| 计次数 | `.count(x)` | ❌ | ❌ |

## 四、可变 vs 不可变（最重要的差异）

```
可变（可增删改）：  列表、字典、集合
不可变（不能改）：  元组（一旦创建，内容固定）
```

**不可变意味着：**
```python
t = (1, 2, 3)
t[0] = 99  # TypeError！元组不能修改
```

**为什么要有元组？**
- 可以作为字典的键（列表不行）
- 可以作为集合的元素（列表不行）
- 表示"这组数据不会变"的语义

```python
# 元组能做键，列表不能
d = {(1, 2): "位置"}  # ✅
d = {[1, 2]: "位置"}  # TypeError: unhashable type: 'list'
```

## 五、解包——所有序列都支持

解包的本质：**把容器里的元素"拆开"赋值给多个变量**，省去手动 `[0]`、`[1]` 的麻烦。

### 基础解包

```python
# 列表解包
a, b, c = [1, 2, 3]

# 元组解包（最常见的场景——函数返回多个值）
def get_user():
    return "张三", 25, "南昌"  # 实际上是返回元组
name, age, city = get_user()

# 字符串解包
a, b, c = "ABC"  # a='A', b='B', c='C'

# 字典解包（拆的是键，不是值！）
a, b = {"x": 1, "y": 2}  # a='x', b='y'
```

### `_` 占位符——忽略不需要的值

```python
name, _, city = get_user()  # 中间 age 不要了
```

### 交换变量——Python 特有的写法

```python
a, b = 1, 2
a, b = b, a  # 交换！a=2, b=1
# 等价于其他语言的 temp = a; a = b; b = temp
```

### `*` 剩余解包——不知道有几个元素时用

```python
first, *middle, last = [1, 2, 3, 4, 5]
# first=1, middle=[2,3,4], last=5

head, *rest = [1, 2, 3, 4]
# head=1, rest=[2,3,4]

*rest, tail = [1, 2, 3, 4]
# rest=[1,2,3], tail=4

first, *_, last = [1, 2, 3, 4, 5]
# first=1, last=5，中间全丢给 _ 忽略
```

### 嵌套解包

```python
data = [1, (2, 3), 4]
a, (b, c), d = data  # a=1, b=2, c=3, d=4

user = {"name": "张三", "scores": [92, 88]}
# 常用于字典 + 列表混合结构的快速取值
```

### `*args` 和 `**kwargs`——函数定义时收包

```python
def func(*args, **kwargs):
    # *args 把传入的位置参数收成元组
    # **kwargs 把传入的关键字参数收成字典
    print(args)   # (1, 2, 3)
    print(kwargs) # {"a": 4, "b": 5}

func(1, 2, 3, a=4, b=5)
```

### 函数调用时解包——和定义时对称

```python
def add(a, b, c):
    return a + b + c

# * 解包列表/元组 → 位置参数
nums = [1, 2, 3]
add(*nums)  # 等价 add(1, 2, 3)

# ** 解包字典 → 关键字参数
info = {"a": 1, "b": 2, "c": 3}
add(**info)  # 等价 add(a=1, b=2, c=3)
```

**对称关系：**
```
定义时：def f(*args, **kwargs)   ← 收包（把多个参数收成一个变量）
调用时：f(*list, **dict)         ← 解包（把一个容器拆成多个参数）
```

### 解包是传引用，不是拷贝

```python
inner = [1, 2]
outer = [3, 4]
data = [inner, outer]

a, b = data       # a = inner 的地址，b = outer 的地址
a.append(99)
print(inner)      # [1, 2, 99]  ← 原来那个列表也被改了
```

要独立副本需显式 `.copy()` 或 `deepcopy()`。

## 六、选型决策：该用哪个？

> 定义数据结构也可用 [[04-listcomp-class/笔记_dataclass_vs_lombok|@dataclass]] 和 pydantic，见模块 07-pydantic。

```
需要记录"第几个"？
  ├── 需要修改 → 列表
  └── 不需要修改 → 元组

需要按键查找？
  ├── 只需要去重/集合运算 → 集合
  └── 需要存取值 → 字典
```

**实战经验：**
- 函数返回多个值 → 元组（隐含"这组数据不会变"）
- 动态列表 → 列表
- 去重/取交集并集 → 集合
- 按名字查东西 → 字典
- 不可变序列→可哈希（做键、进集合） → 元组

## 七、zip + sorted 组合排序（实战高频）

**核心原则：** 多个关联列表不能单独排序，必须打包后再整体排序。

### 基本用法

```python
names = ["王五", "赵六", "孙七"]
scores = [78, 95, 82]
grades = ["C", "A", "B"]

# 3 步：打包 → 按分数排序 → 使用
sorted_data = sorted(zip(names, scores, grades), key=lambda x: x[1])

for name, score, grade in sorted_data:
    print(f"{name}: {score}分 ({grade})")
```

**执行过程拆解：**

1. `zip(names, scores, grades)` → `[("王五",78,"C"), ("赵六",95,"A"), ("孙七",82,"B")]`
2. `sorted(..., key=lambda x: x[1])` — 按每个元组的第 2 个元素（分数）升序排列
3. 遍历结果，解包赋值给 `name, score, grade`

### 降序排列

```python
sorted_data = sorted(zip(names, scores, grades), key=lambda x: x[1], reverse=True)
```

### 多条件排序

```python
# 先按分数升序，分数相同再按姓名排序
sorted_data = sorted(zip(names, scores, grades), key=lambda x: (x[1], x[0]))
```

`key` 返回元组 `(分数, 姓名)`，Python 会先比第一个，相同再比第二个。

### 排序后转字典

```python
sorted_dict = {name: score for name, score, grade in sorted_data}
```

### ❌ 常见错误：单独排序 scores

```python
scores_sorted = sorted(scores)  # 分数变了，但姓名和等级还是原顺序
for name, score, grade in zip(names, scores_sorted, grades):
    # 对应关系全乱了！
    print(f"{name}: {score}分 ({grade})")
```

**教训：只要涉及多个关联列表，先 zip 打包再整体操作。**

## 八、浅拷贝 vs 深拷贝（引用机制）

解包是传引用，`=` 也是传引用。想要独立副本必须显式拷贝：

```python
a = [1, [2, 3]]
b = a              # 引用：b 改 a 也会改
c = a.copy()       # 浅拷贝：c[0] 是新的，但 c[1] 还是引用的同一个 [2,3]
d = deepcopy(a)    # 深拷贝：全部独立
```

| 方式 | 第一层独立 | 嵌套层独立 |
|------|-----------|-----------|
| `b = a` | ❌ | ❌ |
| `c = a.copy()` / `list(a)` / `a[:]` | ✅ | ❌ |
| `from copy import deepcopy` | ✅ | ✅ |

**实战判断：** 如果数据结构只有一层（全是值没有嵌套），浅拷贝够用。有嵌套就要 `deepcopy`。

## 八、排序

```python
nums = [3, 1, 4, 1, 5]

# 列表用 .sort() — 原地排序，修改原列表
nums.sort()
nums.sort(reverse=True)
nums.sort(key=lambda x: -x)  # 按自定义规则

# sorted() — 返回新列表，原数据不变（所有数据结构通用）
sorted_nums = sorted(nums)
sorted_nums = sorted(nums, reverse=True)

# 字典排序
d = {"b": 2, "a": 1, "c": 3}
sorted(d)                     # ['a', 'b', 'c'] — 按键排序（只返回键）
dict(sorted(d.items()))       # {'a': 1, 'b': 2, 'c': 3} — 按键排序
sorted(d.items(), key=lambda x: x[1])  # [('a',1), ('b',2), ('c',3)] — 按值排序
```

## 九、遍历技巧

```python
d = {"a": 1, "b": 2, "c": 3}

# 遍历键
for k in d:          # 默认就是遍历键
for k in d.keys():   # 显式写法

# 遍历值
for v in d.values():

# 同时遍历键和值
for k, v in d.items():
    print(k, v)

# 带索引遍历（enumerate 和任何序列搭配）
names = ["张三", "李四", "王五"]
for i, name in enumerate(names, start=1):
    print(f"第{i}位：{name}")
```

## 十、集合运算

集合的真正价值不只是去重，是做集合运算：

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b   # 并集：{1,2,3,4,5,6}
a & b   # 交集：{3,4}
a - b   # 差集（在a不在b）：{1,2}
a ^ b   # 对称差（不在交集的）：{1,2,5,6}
a <= b  # 子集判断
a >= b  # 超集判断
```

**实战场景：** 判断两个列表是否有重叠元素、权限检查、标签筛选。

## 十一、常见陷阱

### 1. 默认参数用列表

```python
def add_student(name, students=[]):  # 问题！列表只创建一次
    students.append(name)
    return students

print(add_student("张三"))  # ['张三']
print(add_student("李四"))  # ['张三', '李四'] — 不是预期的 ['李四']
```
**修复：** 用 `None` 做默认值，内部创建

```python
def add_student(name, students=None):
    if students is None:
        students = []
    students.append(name)
    return students
```

### 2. 循环时删除元素

```python
nums = [1, 2, 3, 4, 5]
for x in nums:
    if x % 2 == 0:
        nums.remove(x)  # 会导致元素错位
```
**修复：** 用新列表收集要保留的，或遍历副本

### 3. 字典的键必须不可变

```python
d = {[1, 2]: "value"}  # TypeError
d = {(1, 2): "value"}  # 元组可以
```

### 4. 字典在 Python 3.7+ 有序

```python
# 3.7+ 字典保持插入顺序（3.6 是 CPython 实现细节）
d = {"b": 1, "a": 2}
list(d)  # ['b', 'a']
```

### 5. 集合不支持索引

```python
s = {1, 2, 3}
s[0]  # TypeError: 'set' object is not subscriptable
```

## 十二、性能速查

| 操作 | 列表 | 字典/集合 |
|------|------|-----------|
| 取值 | O(1) 按索引 | O(1) 按键/哈希 |
| 查找是否存在 | O(n) 遍历 | O(1) 哈希 |
| 追加 | O(1) 尾部 | O(1) |
| 插入 | O(n) 要移动 | — |
| 删除 | O(n) 找 + O(1) 删 | O(1) 哈希删 |

**实战结论：** 大量查找用字典/集合，按顺序存数据用列表。元组可哈希，不可变。

## 十三、结构互转：四者全通

### 速查总表

> "from" 是源头，"to" 是目标，交叉格就是转换方法。

| from → to | list | tuple | set | dict |
|:---:|------|-------|-----|------|
| **list** | — | `tuple(lst)` | `set(lst)` | `dict(pairs)` ① |
| **tuple** | `list(t)` | — | `set(t)` | `dict(pairs)` ① |
| **set** | `list(s)` | `tuple(s)` | — | `dict.fromkeys(s)` ② |
| **dict** | `list(d)` ③ | `tuple(d)` ③ | `set(d)` ③ | — |

> ① 列表/元组里必须是 `(键, 值)` 对才能转字典  
> ② `fromkeys` 键来集合，值统一（要不同值用推导式）  
> ③ 字典直接转得到的是**键**的列表/元组/集合，值要单独取

---

### 列表 ↔ 元组

```python
lst = [1, 2, 3]
tuple(lst)   # (1, 2, 3)

t = (1, 2, 3)
list(t)      # [1, 2, 3]
```

### 列表/元组 ↔ 集合（自动去重）

```python
lst = [1, 2, 1, 3, 2]
set(lst)     # {1, 2, 3}  ← 去重了

s = {1, 2, 3}
list(s)      # [1, 2, 3]  ← 顺序不保证
tuple(s)     # (1, 2, 3)
```

### 列表/元组 → 字典（必须是键值对）

```python
# 只有这种结构能转字典
pairs = [("name", "tom"), ("age", 18)]
dict(pairs)            # {'name': 'tom', 'age': 18}

# zip 帮你造键值对
keys = ["name", "age"]
vals = ["tom", 18]
dict(zip(keys, vals))  # {'name': 'tom', 'age': 18}

# 这种不能转——不是键值对
lst = [1, 2, 3]
# dict(lst)  # ❌ TypeError
```

### 字典 → 列表/元组/集合

```python
d = {"name": "tom", "age": 18}

# 取键
list(d)          # ['name', 'age']
tuple(d)         # ('name', 'age')
set(d)           # {'name', 'age'}

# 取值
list(d.values()) # ['tom', 18]

# 取键值对
list(d.items())  # [('name', 'tom'), ('age', '18')]
```

### 集合 → 字典

```python
s = {"name", "age"}

# fromkeys：所有键配同一个值
dict.fromkeys(s, "")    # {'name': '', 'age': ''}
dict.fromkeys(s, 0)     # {'name': 0, 'age': 0}

# 要不同值用推导式
{s: len(s) for s in ["apple", "banana"]}  # {'apple': 5, 'banana': 6}
```

### zip 打包 ↔ zip 拆包

```python
names = ["tom", "jack"]
scores = [124, 116]
pairs = list(zip(names, scores))   # [('tom', 124), ('jack', 116)]

# 拆回去
names2, scores2 = zip(*pairs)
list(names2)   # ['tom', 'jack']
list(scores2)  # [124, 116]
```

### 一图全览

```text
list ── tuple() ──► tuple
list ◄── list()  ──  tuple

list ── set() ────► set   (去重)
list ◄── list() ─── set   (顺序不保证)

list ── dict(pairs) ──► dict   (必须 [(k,v),...])
list ◄── list(d) ───── dict   (取键)

set ── dict.fromkeys(s) ──► dict
dict ◄── set(d) ───────── set   (取键)

dict ── .items() ──► [(k,v),...] ── dict() ──► dict
```
