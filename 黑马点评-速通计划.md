### Day 2：读 PDF 精华 + 提炼模式（2h）

这天的重点是**把 PDF 里的技术方案变成你自己的理解**。

#### 上午：阅读以下页面

| 页码 | 内容 | 为什么重要 |
|------|------|-----------|
| 10-12 | 简历怎么写项目 | 理解"写在简历上的每一点都要有话术"这个原则 |
| 13-15 | 缓存击穿话术 | **最核心的话术模板**，学会"背景→问题→方案→复盘"四步法 |
| 16-17 | 缓存穿透话术 | 同上，补全缓存三大问题（击穿/穿透/雪崩） |
| 18-19 | 数据一致性话术 | 更新数据库后删缓存 + MQ 重试 + TTL 兜底 |

#### 下午：提炼技术模式

**读完这几十页，你要能回答三个问题**：

```
Q1: 缓存击穿怎么解决？
    → 互斥锁（一致性优先）vs 逻辑过期（可用性优先）
    → 本项目的选择：逻辑过期，因为点评是社交媒体场景，不强求一致性

Q2: 缓存穿透怎么解决？
    → 缓存空值（简单但有内存消耗，用 TTL 解决）
    → 布隆过滤器（复杂、有误判、不能删除数据）
    → 本项目的选择：缓存空值，因为商铺数量不大

Q3: 数据库和缓存数据不一致怎么办？
    → 先更新 DB → 删缓存 → 删除失败则 MQ 重试 → TTL 兜底
    → 为什么不是"先删缓存再更新 DB"？（并发时更容易出问题）
    → 为什么不是"延迟双删"？（延迟时间不好定）
```

**这三个 Q&A 是你面试时能讲的核心弹药。**

### Day 3：秒杀模块 + 面试话术背诵（2h）

#### 上午：秒杀模块

| 页码 | 内容 |
|------|------|
| 秒杀部分 | 优惠券秒杀流程、库存扣减、一人一单、分布式锁 |

**核心理解**：
```
秒杀流程：
  用户抢券 → 检查库存（Redis 预减） → 检查是否已买过
    → 异步下单（MQ） → 扣减 MySQL 库存 → 返回结果

关键问题：
  - 怎么防止超卖？ Redis 预减库存 + 数据库乐观锁
  - 怎么防止一人多抢？ Redis setnx 分布式锁
  - 性能不够怎么办？ 多级缓存（Caffeine + Redis）
```

#### 下午：开口练

把 Day 2 提炼的三个 Q&A + 秒杀流程，**对着空气讲出来**。不是默念，是出声讲。

要求：
- 每个问题讲 1-2 分钟
- 要求流畅、有逻辑（背景→问题→方案→复盘）
- 录一遍自己听，看哪里卡壳

## 四、怎么内化——把黑马点评变成你自己的东西

### 1. 复制话术结构，不复制话术文字

PDF 里的话术是作者写的，你要内化的是**框架**：

```
背景阐述 → 这个功能面临什么场景？有什么特点？
问题剖析 → 如果不处理，会出什么问题？
方案构思 → 有哪些可选方案？各有什么优缺点？
复盘总结 → 我为什么选了方案A而不是B？实际效果如何？
```

**这个四步框架，适用你简历上任何一个项目点。** 苍穹外卖、AI-Meeting、你自己的 AI 项目——用同一个框架讲。

### 2. 把黑马点评的技术点迁移到苍穹外卖

```
苍穹外卖里也有缓存场景：
  - 菜品信息 → 读多写少 → 适合 Redis 缓存 → 要考虑数据一致性
  - 热门菜品 → 热点数据 → 要考虑缓存击穿
  - 套餐秒杀（如果有） → 和黑马点评秒杀一样的模式

面试时你可以说：
  "在做苍穹外卖时，我对菜品查询模块加了 Redis 缓存，
   参考了黑马点评项目的缓存方案，使用了逻辑过期解决热点数据击穿问题..."
```

**两个项目互相引用，面试官会觉得你真的理解了，而不是复制粘贴。**

### 3. 把 Redis 八股变成项目经验

PDF 后半部分（100 页后）是纯八股——Redis 数据类型、持久化、集群、Kafka 原理等。**不要从头读到尾**，而是：

- 当你项目话术中提到"缓存击穿" → 翻到对应八股页，理解底层的 Redis 原理
- 当你项目话术中提到"消息队列重试" → 翻到 Kafka 八股，理解消息不丢失/不重复
- 以项目为线索串八股，不要孤立背

## 六、最重要的提醒

`★ Insight ─────────────────────────────────────`
- **这份 PDF 最大的价值不是"黑马点评"这个项目本身，而是它教你"怎么把项目讲给面试官听"。** 你之前苍穹外卖知道做了什么但不知道怎么讲，这份 PDF 的话术模板就是答案。
