## 三、核心差异：不只是"有没有"，是"怎么教的"

### 1. 原笔记：问题驱动，直接给答案

```
"问题来了：acc.password 谁都看得见
→ Python 没有 private
→ 用 _ 和 __ 约定解决"
```

特点：快速、实用、适合**你已经知道要什么**的时候检索。

### 2. CS50P：故事驱动，带你走弯路

```
tuple → 不能改 → 换 list → 没语义 → 换 dict → 
功能不够 → 引出 class → 但绕过验证 → 引出 @property
```

特点：慢、啰嗦，但让你**理解 WHY**。适合**第一次学**或者**理解不深**时回看。

### 3. Java 对比：CS50P 的系统性优势

原笔记偶尔提 Java，但 CS50P **每讲末尾都有完整的 Java vs Python 对比表**。这对你"从 Java 迁移到 Python"的背景特别有价值。

## 四、你应该如何看待和使用这两套笔记

### 不要做的事

- ❌ 不要试图把两套笔记"合并"——它们是互补的，不是竞争的
- ❌ 不要全看一遍——26 个文件 + 11 个文件 = 太多了
- ❌ 不要纠结"哪个版本更好"——不同场景用不同版本

### 应该做的事

```
📖 学新概念时 → 看 CS50P 笔记
   （因为它的渐进式教学让你理解 WHY）
   
🔍 复习/查语法时 → 看原笔记
   （因为它的细粒度让你快速定位）
   
🔗 需要 Java 对照时 → 看 CS50P 笔记末尾的对比表
   
🐛 遇到 bug 时 → 看原笔记里的"常见错误"章节
   
🧪 要动手练时 → 用原笔记的 .py 练习文件
```

### 实际的学习路径建议

```
第1遍（现在）: 看 CS50P 笔记的 09-OOP（理解 Python OOP 的 WHY）
            + 看 26-unit-tests（全新领域）
            + 看 27-regex（全新领域）
            
第2遍（暑假）: 把 26-unit-tests 和 27-regex 的 practice TODO 写完
            把 03/05/16/23 的 cs50p_supplement.md 过一遍
            
日常维护: 原笔记是你的"主仓库"
        CS50P 笔记是你的"课外参考书"
        新学的任何东西 → 更新原笔记
```
