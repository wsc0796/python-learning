## 一、普通人如何正确参与 AI 风口

### 核心认知

**不一定要训练大模型。** 训练一个大模型需要：
- 数千张 GPU
- 海量高质量数据
- 专业的分布式训练团队
- 数百万美元起步

普通人更现实的参与方式：

```
学会使用大模型 → 学会设计 Prompt → 学会搭建智能体
    → 学会工作流自动化 → 结合行业场景做应用
```

**扣子（Coze）解决的就是"让普通开发者能搭 AI 应用"这个问题。**

### 类比

| 角色 | 做什么 | 类比 |
|------|--------|------|
| 模型研究员 | 训练新模型 | 造发动机 |
| AI 应用开发者 | 用模型搭应用 | 造车 |
| 扣子用户 | 拖拽搭工作流 | 组装车 |

## 二、扣子平台核心架构

```
空间 (Space)
  └── 项目 (Project)
        ├── 智能体 (Bot / Agent)
        │     ├── 人设与 Prompt
        │     ├── 插件 (Plugin)
        │     ├── 知识库 (Knowledge Base)
        │     └── 工作流 (Workflow)
        ├── 工作流 (独立)
        └── 资源库 (Resource)
              ├── 知识库
              ├── 插件
              └── 变量/模板
```

| 概念 | 一句话 | 类比 |
|------|--------|------|
| **空间** | 你的工作台，隔离不同项目 | GitHub Organization |
| **项目** | 一个具体的 AI 应用 | GitHub Repository |
| **智能体** | 有特定人设 + 能力的 AI 助手 | 一个员工 |
| **工作流** | 多步骤自动化任务链条 | 工厂流水线 |
| **插件** | 外部工具/API | 微服务 |
| **知识库** | 上传的文档/数据，让 AI 能"知道"你的业务 | 企业 Wiki |

## 三、可视化流程编排的核心价值

```text
传统开发：
  写代码 → 调 API → 处理 JSON → 写代码 → 调另一个 API → ...

扣子工作流：
  拖拽节点 → 连线 → 配置参数 → 跑通
```

**核心价值**：
1. **降低门槛** — 不需要写后端代码就能搭 AI 应用
2. **拆解复杂任务** — 把一个大任务拆成多个小节点
3. **数据可视化传递** — 可以看到数据在节点间怎么流动
4. **方便调试** — 每个节点可以单独测试
5. **可复用** — 工作流可以复制、分享、绑定到不同智能体

## 四、工作流适合/不适合的场景

### 适合工作流

- 多步骤任务（先 A 再 B 再 C）
- 格式要求固定的任务（模板化输出）
- 需要调用外部工具的任务（查天气 + 生成文案）
- 需要查询知识库的任务（客服问答）
- 批量处理任务（批量生成标题/海报）
- 输出结构化结果的任务（JSON 格式回复）

### 不适合工作流

- 单轮简单对话（直接用智能体就行）
- 需要复杂逻辑判断的任务（扣子代码块有限制）
- 实时性要求极高的任务
- 需要大量计算的任务

## 五、与你的技术栈的关系

`★ Insight ─────────────────────────────────────`
- 扣子工作流和你学的 FastAPI 不是竞争关系，是**互补关系**。扣子适合快速验证想法、搭原型、做内部工具。当真需要自定义逻辑、对接自己的数据库、做复杂业务时，用 FastAPI 自己写后端，把 LLM 当作一个 API 来调。
- 工作流的"节点 → 连线 → 数据传递"模型，和你写代码时的"函数 → 调用 → 返回值"本质上是同一件事。你在扣子里搭工作流的经验，反过来会让你更理解怎么写可组合的函数。
- 扣子是目前国内对开发者最友好的 AI 应用搭建平台之一，免费额度够学习和做小项目。这个专题学完，你简历上可以写"熟练使用 Coze 平台搭建 AI 工作流"——对就业方向这是加分项。
