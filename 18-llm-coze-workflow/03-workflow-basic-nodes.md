# 03. 工作流核心组成要素

> 前置：[[02-coze-core-concepts]]
> 目标：理解每种节点的作用和适用场景
> 用时：约 12 分钟
>
> 相关笔记：[[05-workflow-entry-practice]] · [[06-advanced-workflow-nodes]]

---

## 一、工作流 = 节点 + 连线

```
[开始] → [LLM处理] → [插件补充] → [代码格式化] → [回复]
                ↘           ↙
              [条件判断] → [不同分支]
```

每个节点是一个**独立的功能单元**，连线代表**数据流向**。

---

## 二、核心节点一览

### 1. 开始节点

**作用**：接收外部输入，定义工作流的入口参数。

```text
常见输入参数：
- user_input: 用户原始输入
- topic: 主题
- city: 城市
- keyword: 关键词
- style: 风格
- count: 数量
```

### 2. LLM 节点

**作用**：调用大模型生成内容。这是工作流的"大脑"。

核心配置：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 模型选择 | 用哪个大模型 | 豆包、GPT、Claude |
| 系统指令 | 设定 AI 的角色和规则 | "你是一名专业文案写手" |
| 用户输入 | 要处理的内容 | `{{开始节点.user_input}}` |
| 输出格式 | 期望的输出结构 | JSON、纯文本、Markdown |
| 温度 | 创造性程度（0-1） | 0.3=严谨，0.8=创意 |

### 3. 插件节点

**作用**：调用外部工具/API，获取实时数据。

常见插件类型：
- 天气查询 → 获取实时天气
- 搜索引擎 → 获取网络信息
- 图片生成 → 调用 Stable Diffusion / DALL-E
- HTTP 请求 → 调任意第三方 API
- 数据库查询 → 读取业务数据

### 4. 代码块节点

**作用**：写 Python/JS 代码处理复杂逻辑。

适用场景：
- JSON 数据清洗和格式转换
- 字符串拼接/截取
- 列表循环处理
- 数据校验
- 复杂计算

```python
def main(input_data: dict) -> dict:
    title = input_data.get("title", "")
    content = input_data.get("content", "")
    return {"markdown": f"# {title}\n\n{content}"}
```

### 5. 条件分支节点

**作用**：根据条件走不同路径。

```text
用户输入
    ↓
判断意图
    ├── "查天气" → 调用天气插件 → 回复天气信息
    ├── "写文案" → 调用 LLM 节点 → 回复文案
    └── "其他"   → 回复引导信息
```

### 6. 循环节点

**作用**：对列表中的每一项重复执行相同操作。

适用场景：
- 批量生成标题（给 10 个主题 → 生成 10 个标题）
- 批量处理商品信息
- 逐条审核合同条款

### 7. 回复节点

**作用**：把最终结果输出给用户。

支持回复格式：
- 纯文本
- Markdown（标题、列表、加粗）
- 消息卡片（飞书/豆包适配）
- 图片/视频 URL

## 三、数据传递逻辑

```text
用户: "帮我写一篇北京周末游攻略"
    ↓
开始节点
  user_input = "北京周末游攻略"
    ↓
LLM 节点
  输入: {{开始节点.user_input}}
  系统指令: "你是旅游博主，生成 Markdown 格式攻略"
  输出: draft = "## 北京周末游攻略\n..."
    ↓
插件节点（天气）
  输入: city = "北京"
  输出: weather = {"today": "晴 26°C", "tomorrow": "多云 22°C"}
    ↓
回复节点
  组合: {{LLM节点.draft}} + {{插件节点.weather}}
  输出: Markdown 格式完整攻略
```

---

## 四、同步运行 vs 异步运行

| 类型 | 含义 | 适用场景 |
|------|------|---------|
| **同步** | 等当前节点完成，再执行下一个 | 简单流程、短耗时任务 |
| **异步** | 发起任务后不等待，继续往下走 | 图片生成、视频生成、长耗时 API |

你在 [[03-async-await]] 里学的 `async/await`，概念完全一样——扣子只是把它做成了可视化的配置项。

---

## 五、节点选择决策树

```text
需要接收用户输入？              → 开始节点
需要 AI 生成内容？              → LLM 节点
需要查外部数据/API？           → 插件节点
需要写代码处理复杂逻辑？        → 代码块节点
需要根据条件走不同流程？        → 条件分支节点
需要重复处理列表里每一项？      → 循环节点
需要输出结果给用户？            → 回复节点
```
