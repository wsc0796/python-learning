## 一、变量：工作流中的数据容器

### 什么是变量

在工作流中，变量是**临时保存数据的容器**。每个节点处理数据，产出数据，数据在节点之间以"变量"的形式传递。

```
用户输入 "帮我写一篇北京旅游文案"
    ↓
开始节点: user_input = "帮我写一篇北京旅游文案"
    ↓
LLM 节点: 读取 user_input → 生成 draft = "北京，一座千年古都..."
    ↓
插件节点: 读取 city = "北京" → 返回 weather = {"condition": "晴", "temp": 26}
    ↓
回复节点: 组合 draft + weather → 最终输出
```

### 常见变量类型

| 类型 | 示例 | 来源 |
|------|------|------|
| 用户输入 | `{{input}}` | 开始节点 |
| LLM 输出 | `{{llm_node.output}}` | LLM 节点 |
| 插件返回 | `{{plugin_node.result}}` | 插件节点 |
| 代码块返回 | `{{code_node.data}}` | 代码块节点 |
| 中间变量 | 你自定义的 | 任意节点 |

## 二、JSON：工作流中的数据格式

**JSON 是工作流中节点之间传递数据的标准格式。** 每个节点的输出几乎都是 JSON。

### JSON 对象（字典）

```json
{
  "city": "北京",
  "weather": "晴",
  "temperature": 26
}
```

访问方式：`{{node_name.city}}` → `"北京"`

### JSON 数组（列表）

```json
["标题1", "标题2", "标题3"]
```

访问方式：`{{node_name[0]}}` → `"标题1"`

### 嵌套 JSON

```json
{
  "city": "北京",
  "forecast": {
    "today": {"condition": "晴", "temp": 26},
    "tomorrow": {"condition": "多云", "temp": 22}
  }
}
```

访问方式：`{{node_name.forecast.today.temp}}` → `26`

### 你在 Python 中已经会了

```python
# Python 字典 → JSON 对象
data = {"city": "北京", "weather": "晴"}
print(data["city"])  # 北京

# Python 列表 → JSON 数组
titles = ["标题1", "标题2", "标题3"]
print(titles[0])    # 标题1
```

**扣子工作流里的 JSON 操作，和你 Python 里操作 dict/list 是同一回事。**

## 三、输入参数和输出参数

每个节点都有：

```
┌─────────────────────────┐
│       节点名称           │
│                         │
│  输入参数：              │
│    - param1: 从哪取数据  │
│    - param2: 从哪取数据  │
│                         │
│  处理逻辑（节点内部）     │
│                         │
│  输出参数：              │
│    - output1: 产出的数据 │
│    - output2: 产出的数据 │
└─────────────────────────┘
```

**理解要点**：
- 一个节点的**输出**，就是下一个节点的**输入来源**
- 节点之间通过**引用变量**来传递数据
- 变量引用的格式通常是 `{{节点名.输出字段名}}`

## 四、正确学习路径

```
1. 理解工作流整体结构    → 知道有哪些节点
2. 学开始节点和结束节点   → 数据从哪来、到哪去
3. 学 LLM 节点           → 大模型怎么被调用
4. 学插件节点            → 外部工具怎么接入
5. 理解变量传递          → 节点间怎么传数据
6. 学代码块节点          → 写 Python 处理复杂逻辑
7. 学条件分支/循环       → 流程控制和批量处理
8. 学异步/多工作流串联    → 复杂场景编排
```

**不要跳步**——很多人卡在"变量引用不对"上，就是因为没先搞懂 JSON 和节点间的数据传递模型。

## 五、常见易错点

| 易错点 | 说明 |
|--------|------|
| 变量路径写错 | `{{node.output}}` 和 `{{node.result}}` 不是一回事 |
| JSON 层级访问错误 | 嵌套 JSON 要逐层访问：`a.b.c` 不是 `a.c` |
| 数组索引从 0 开始 | `list[0]` 是第一个元素，不是 `list[1]` |
| 变量为空 | 上一个节点没正确输出，下一个节点自然取不到 |
| 忘记配置输出参数 | 节点处理完了，但没"暴露"输出，下游拿不到 |
