# 09. 多类型工作流实战项目

> 前置：[[08-industry-cases]]
> 目标：有具体的项目思路可以照着做
> 用时：约 8 分钟（阅读项目思路）+ 实践时每个项目 30-60 分钟

---

## 项目一：小红书主题文案 + 配图

### 工作流结构

```
开始节点
  ├── topic: 主题（如"春季护肤"）
  └── count: 数量（如 3）
    ↓
LLM 节点（文案）
  系统指令: "你是小红书博主，风格活泼、有emoji、分点列举"
  输出: 文案文本
    ↓
代码块节点
  提取文案中的关键词作为图片 prompt
  输出: prompt_list (数组)
    ↓
循环节点（对每个 prompt）
  图片生成插件
    ↓
回复节点: 文案 + 图片
```

## 项目二：英语单词学习卡片

### 工作流结构

```
开始节点
  ├── word: 单词（如 "serendipity"）
  └── level: 难度
    ↓
LLM 节点
  系统指令: "生成英语单词学习卡片，包含：单词、音标、中英文释义、例句、词根词缀、记忆技巧"
  输出: 卡片内容
    ↓
代码块节点
  格式化为 Markdown 卡片
    ↓
回复节点: Markdown 格式卡片
```

## 项目三：城市文旅视频脚本

### 工作流结构

```
开始节点
  └── city: 城市名
    ↓
LLM 节点（攻略）
  输出: 旅游攻略文本
    ↓
代码块节点
  提取关键景点和描述 → prompt_list
    ↓
循环节点（对每个景点）
  LLM 生成图片 prompt → 图片生成插件（异步）
    ↓
LLM 节点（视频脚本）
  根据攻略 + 图片 生成分镜脚本
    ↓
回复节点: 完整的视频脚本 + 配图
```

## 项目四：电商海报生成

### 工作流结构

```
开始节点
  ├── product_name: 商品名
  ├── selling_points: 卖点
  └── style: 风格（简约/促销/高端）
    ↓
LLM 节点（图片 prompt）
  系统指令: "你是电商设计师，根据商品信息和风格生成图片 prompt"
  输出: image_prompt
    ↓
LLM 节点（海报文案）
  输出: 主标题 + 副标题 + 促销文案
    ↓
插件节点（图片生成，异步）
    ↓
回复节点: 海报图 + 文案
```

## 项目五：法务合同审核助手

### 工作流结构

```
开始节点
  └── contract_text: 合同全文
    ↓
LLM 节点（条款提取）
  系统指令: "提取合同中的关键条款"
  输出: 条款列表 (数组)
    ↓
循环节点（对每条条款）
  LLM 节点（风险评估）
    系统指令: "你是法务顾问，评估该条款的风险"
    输出: 风险评级 + 建议
    ↓
代码块节点
  汇总所有条款的风险评级
  输出: 风险汇总表
    ↓
回复节点: Markdown 格式的审核报告
```

## 项目六：批量财务发票助手

### 工作流结构

```
开始节点
  └── invoice_data: 发票信息（JSON）
    ↓
代码块节点
  解析 JSON → 提取关键字段（金额、税号、日期）
  验证字段完整性
    ↓
条件分支
  ├── 验证通过 → LLM 节点生成发票摘要
  └── 验证失败 → 回复"缺少必要字段: XXX"
    ↓
回复节点: 发票处理结果
```

## 项目选择建议

| 你的方向 | 先做 |
|---------|------|
| 想展示 AI 能力 | 项目一（小红书文案）— 直观、好展示 |
| 想对接后端 | 项目六（发票助手）— 需要代码块+HTTP |
| 想做内容产品 | 项目三（文旅视频脚本）— 可扩展性强 |
| 想做企业服务 | 项目五（合同审核）— 有付费意愿的场景 |
