## 一、GPT 系列发展脉络

| 模型 | 年份 | 参数规模 | 核心贡献 |
|------|------|---------|---------|
| GPT-1 | 2018 | 1.17 亿 | 证明了"预训练 + 微调"范式有效 |
| GPT-2 | 2019 | 15 亿 | 展示了大规模文本生成能力（一度因担心滥用未开源） |
| GPT-3 | 2020 | 1750 亿 | 发现 **Few-shot** 能力——不用微调，给几个例子就能做新任务 |

## 二、GPT 的核心思想

### 预训练（Pre-training）

在海量互联网文本上，用"预测下一个 token"的方式训练。

```
输入: "中国的首都是"
模型预测: "北京" (概率最高)
实际标签: "北京"
→ 如果预测错了，调整参数
```

这个过程重复数万亿次，模型逐渐学会了语法、常识、推理。

### 但是问题来了

GPT-3 虽然很强，但它的回答方式是这样的：

```
问: "1+1等于几?"
答: "1+1等于2。那么1+2等于几?"
   ↑ 它在续写，不是在回答问题
```

**它能生成文本，但不一定会"对话"。** 这就是为什么需要 ChatGPT。

## 三、从 GPT-3 到 ChatGPT：核心是 RLHF

### 第一步：监督微调（SFT）

人工标注"好的回答"：
- 标注员写：用户问 X，好的回答是 Y
- 用这些高质量数据微调 GPT-3
- 模型学会了"问答格式"

### 第二步：训练奖励模型（RM）

对同一个问题生成多个回答，让标注员**排序**（哪个更好，不是打分）：

```
问: "怎么学Python?"

回答A: "看文档"            → 排名 4（最差）
回答B: "推荐CS50P课程..."  → 排名 1（最好）
回答C: "多做练习..."       → 排名 2
回答D: "买本书..."         → 排名 3
```

用这些排序数据训练一个"奖励模型"——它能给任意回答打分。

### 第三步：强化学习优化（PPO）

```
生成回答 → 奖励模型打分 → 高分=加强，低分=削弱 → 继续生成 → ...
```

反复循环，模型逐渐学会生成**人类偏好的回答**——有帮助、准确、安全、有礼貌。

**这就是 ChatGPT"会说人话"的秘密。**

## 四、Few-shot：不给训练，只给例子

```text
Prompt:
  把以下词语翻译成英文：
  苹果 → apple
  香蕉 → banana
  橘子 →

模型输出: orange
```

模型**没有**被重新训练，只是根据前面的例子"理解"了任务格式。

| 术语 | 含义 |
|------|------|
| Zero-shot | 不给例子，直接问 |
| One-shot | 给 1 个例子 |
| Few-shot | 给几个例子 |

GPT-3 最大的突破就是 **Few-shot 能力强**——给几个例子就能做新任务，不需要额外训练。

---

## 五、ChatGPT 训练流程（完整版）

```
第0步: 大规模预训练（海量文本，预测下一个token）
    ↓
第1步: 监督微调 SFT（人工标注问答对，教会模型"对话格式"）
    ↓
第2步: 训练奖励模型 RM（收集人类偏好排序数据，训练"打分员"）
    ↓
第3步: 强化学习 PPO（奖励模型引导模型生成人类偏好的回答）
    ↓
ChatGPT: 一个"会说人话"、有帮助、安全的对话模型
```

## 六、为什么这些对你重要

`★ Insight ─────────────────────────────────────`
- **RLHF 是整个 AI 对齐（Alignment）领域的核心概念**。理解了 RLHF，你就理解了为什么 Claude 比 GPT-3 更安全、为什么国内大模型也在做 RLHF。这不只是技术问题，也是"怎么让 AI 听人话"的问题。
- **Few-shot 思维在做 Agent 时最有用**。你不会训练自己的 LLM，但你会在 Prompt 里给例子让模型理解格式。Few-shot Prompting 是 AI Agent 开发者的基础技能。
- 了解这个训练流程，你就不会问"为什么 ChatGPT 不知道 2024 年的事"——因为预训练数据有截止日期，且 RLHF 阶段只是调整行为偏好，不注入新知识。
