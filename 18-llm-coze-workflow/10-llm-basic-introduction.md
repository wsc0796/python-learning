# 10. LLM 基础知识

> 用时：约 12 分钟

## 一、什么是 LLM

**LLM = Large Language Model = 大语言模型**

一句话定义：

> 根据已有上下文，预测下一个最可能出现的 token（词元）的数学模型。

你输入 "今天天气真"，LLM 算出下一个 token 最可能是 "好"（概率 85%），其次是 "差"（10%），"热"（3%）...

把它重复几亿次、配上千亿参数、用海量文本训练，就是你现在用的 ChatGPT、Claude、豆包。

---

## 二、什么是语言模型

语言模型做一件事：**计算一句话出现的概率**。

```
P("我爱学习Python") = ?
P("我学习Python爱") = ?
```

一个好的语言模型会给"正常的话"更高概率，"胡言乱语"更低概率。

### 核心公式（简化理解）

```
P("我爱学习Python")
= P(我) × P(爱|我) × P(学习|我爱) × P(Python|我爱学习)
```

每个词的概率依赖于它前面的所有词。

---

## 三、N-Gram：朴素的近似方法

### 问题

如果依赖前面**所有**词，计算量太大，而且大部分词组在语料库中从未出现过。

### 解决：只看前面 N-1 个词

| 类型 | 看前面几个词 | 示例 |
|------|------------|------|
| Unigram | 0 个（只看自己） | P(Python) |
| Bigram | 1 个 | P(Python \| 学习) |
| Trigram | 2 个 | P(Python \| 爱 学习) |

**Bigram 示例**：

```
P("我爱学习Python")
≈ P(我) × P(爱|我) × P(学习|爱) × P(Python|学习)
```

### N-Gram 的问题

- N 太小 → 丢失长距离依赖
- N 太大 → 数据稀疏，大部分词组没见过
- 完全不理解语义，只是统计词组频率

现代 LLM 用 Transformer 架构替代了 N-Gram，能捕捉任意距离的依赖关系。

## 四、LLM 评价指标

### BLEU（Bilingual Evaluation Understudy）

- **用途**：评估机器翻译质量
- **核心思想**：生成文本和参考答案的 n-gram 重合度
- **值域**：0-1（越高越好）
- **局限**：只看表面匹配，不管语义对不对

```
参考答案: "猫在垫子上"
生成结果: "垫子上的猫"
→ BLEU 得分取决于 n-gram 重叠，但语序不对可能仍然得分较低
```

### ROUGE（Recall-Oriented Understudy for Gisting Evaluation）

- **用途**：评估文本摘要质量
- **核心思想**：参考答案里有多少内容被生成结果覆盖了
- **关注点**：召回率（不要漏重要信息）

### PPL（Perplexity，困惑度）

- **用途**：评估语言模型本身的预测能力
- **核心思想**：模型对测试文本有多"惊讶"
- **越低越好**：PPL=1 表示模型完美预测了每一个词

```
PPL=10 → 平均每个位置有 10 个合理选项
PPL=100 → 模型很不确定，如同抛硬币
```

## 五、Token 是什么

LLM 不是逐字处理的，是逐 **token**。

```
"我喜欢学习Python"
可能被分成: ["我", "喜欢", "学习", "Python"]

"ChatGPT is amazing"
可能被分成: ["Chat", "G", "PT", " is", " amazing"]
```

- 1 个 token ≈ 0.75 个英文单词 ≈ 1-2 个中文字
- 模型的"上下文窗口"（context window）按 token 计，不是字数
- API 计费也按 token 算

## 六、你现在应该记住的

| 概念 | 一句话 |
|------|--------|
| LLM | 根据上文预测下一个 token 的大模型 |
| 语言模型 | 计算一句话概率的模型 |
| N-Gram | 只看前 N-1 个词的简化语言模型 |
| BLEU | 机器翻译的 n-gram 匹配率 |
| ROUGE | 摘要的召回率指标 |
| PPL | 困惑度，越低模型越"不困惑" |
| Token | 模型处理的最小文本单元 |
