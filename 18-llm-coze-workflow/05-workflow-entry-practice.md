# 05. 工作流入门实操

> 前置：[[03-workflow-basic-nodes]] · [[04-coze-platform-setup]]（已注册账号）
> 目标：从零搭建一个完整的 AI 工作流，跑通最小闭环
> 用时：约 15 分钟
>
> 相关笔记：[[06-advanced-workflow-nodes]]

---

## 一、新建工作流步骤

```
1. 登录扣子 → 选择空间
2. 新建项目（或直接在空间里新建工作流）
3. 命名工作流（例："旅游攻略生成器"）
4. 看到空白画布 → 开始拖拽节点
```

---

## 二、配置开始节点

开始节点定义了工作流接收什么输入。

```text
添加输入参数：

参数名: user_input
类型: String
说明: 用户输入的旅游需求

参数名: style
类型: String
说明: 攻略风格（详细/简洁/文艺）
默认值: "详细"
```

---

## 三、配置 LLM 节点

### 模型选择

选择可用的大模型（如豆包、GPT-4o mini）。

### 系统指令（System Prompt）

```text
你是一名经验丰富的旅游博主。
你的任务是：
1. 根据用户需求生成一份旅游攻略
2. 攻略用 Markdown 格式
3. 包含以下板块：景点推荐、美食推荐、交通建议、注意事项
4. 风格要求：{{style}}
5. 字数控制在 500-800 字
```

### 用户输入（User Prompt）

```text
{{开始节点.user_input}}
```

### 输出配置

- 输出变量名：`draft`
- 输出格式：文本

---

## 四、配置回复节点

回复节点把 LLM 生成的 `draft` 返回给用户。

```text
回复内容：
{{LLM节点.draft}}

回复格式：Markdown
```

---

## 五、调试运行

### 测试输入

```text
user_input: "北京周末两日游，喜欢历史和文化"
style: "详细"
```

### 点击运行

观察每个节点的输入/输出：
1. 开始节点 → 看到 user_input 和 style 的值
2. LLM 节点 → 看到生成的 draft 内容
3. 回复节点 → 看到最终输出

### 常见调试技巧

| 问题 | 检查项 |
|------|--------|
| LLM 输出为空 | 系统指令是否太模糊？尝试加具体约束 |
| 格式不对 | 系统指令里是否明确要求了格式？ |
| 变量引用报错 | 检查 `{{节点名.字段名}}` 是否正确 |
| 节点执行超时 | LLM 生成太慢？尝试换更快的模型或减少输出字数 |

---

## 六、发布与使用

工作流调试通过后：
1. 点击"发布"
2. 可以绑定到智能体（Bot）
3. 也可以生成 API 接口供外部调用
4. 也可以分享链接给别人使用

---

## 七、你的第一个工作流：最小检查清单

- [ ] 开始节点至少有一个输入参数
- [ ] LLM 节点配置了系统指令和用户输入
- [ ] 系统指令里有角色设定 + 任务说明 + 格式要求
- [ ] LLM 节点的输入引用了开始节点的变量
- [ ] 回复节点引用了 LLM 节点的输出
- [ ] 用至少 2 组不同的输入参数测试
- [ ] 确认输出格式符合预期
