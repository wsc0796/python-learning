# 程序阅读与改错

## 读程序输出的顺序

1. 先看变量初始值。
2. 再看分支条件是否成立。
3. 循环题画表：第几轮、变量值、是否 break/continue。
4. 函数题先看调用处，再回到函数定义。
5. 类题先看对象创建时 `__init__`，再看方法调用。
6. 异常题先看哪一行出错，再判断走哪个 `except`、是否走 `else/finally`。

## 高频改错点

| 类型 | 常见错误 | 修正方向 |
|---|---|---|
| 语法 | 少冒号、缩进错、括号不配对 | 先让程序能运行 |
| 判断 | 把 `==` 写成 `=` | 判断用 `==` |
| 循环 | range 边界多一位或少一位 | 记住左闭右开 |
| 列表 | 遍历时直接删除元素 | 用新列表保存结果 |
| 字典 | `d[key]` 导致 KeyError | 用 `get` 或先判断 key |
| 函数 | 忘记 return | 明确输出值来自 return 还是 print |
| 作用域 | 函数里修改全局变量未写 `global` | 需要修改才写 `global` |
| OOP | 忘记 self | 实例方法第一个参数是 self |
| 文件 | 写模式覆盖原文件 | 区分 `w` 和 `a` |
| 异常 | `except` 太宽或顺序错误 | 具体异常放前面 |
