## 今日最卡的点

- `input()` 和函数参数混淆：函数已经收到 `a`、`b` 时，不应该再 `input(a)`。
- `print()` 和表达式混淆：`print()` 只是显示，返回值是 `None`，不能拿它继续做加减。
- Java 写法迁移到 Python：Python 变量不用写 `int a = ...`。
- 循环变量混淆：`for i in range(...)` 时，当前轮数字是 `i`，不是固定的 `n`。
- 综合统计里把 `total` 当成当前分数，导致最高/最低分判断错误。

## 重复错误

- 括号位置不稳：`print(int(a)) + int(b)` 应改成先算完再 `print(...)`。
- 分支结构不稳：多个独立 `if` 和互斥的 `if / elif / else` 混用。
- 循环边界不稳：`range(1, n)` 不包含 `n`，需要 `range(1, n + 1)`。
- `break` / `continue` 执行顺序需要继续口述和变式练习。
