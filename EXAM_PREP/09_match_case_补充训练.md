# match/case 跨章节补充训练

定位：补充训练，不是期末主线。考试优先级低于 `if / elif / else`、循环、函数、容器、文件和异常。

本机环境：Python 3.12.10，支持 `match/case`。

## 先修正一个理解

不要记成：

> `match/case` 能做，`if/elif` 做不了。

更准确的说法是：

> `if/elif` 也能实现相同业务逻辑，但 `match/case` 在处理结构化数据时通常更直观、简洁。

## 练习 1：状态码分类器

考点：函数、返回值、类型提示、`|` 多值匹配。

要求写函数：

```python
def classify_status(status: int) -> str:
    ...
```

分类规则：

- `200 | 201 | 204`：成功
- `301 | 302`：重定向
- `400 | 403 | 404`：客户端错误
- `500 | 502 | 503`：服务端错误
- 其他：未知状态

限制：先用 `if / elif / else` 写一遍，再用 `match/case` 改写。

## 练习 2：命令菜单系统

考点：循环、字符串方法、`break`、`case _`。

命令：

```text
add
list
delete
help
quit
```

要求：

1. 使用 `while True` 持续处理命令。
2. 使用 `.strip().lower()` 规范输入。
3. 使用 `match` 分发命令。
4. 输入 `quit` 时 `break`。
5. 未知命令用 `case _` 处理。

注意：这是循环和字符串复习，不是项目开发。

## 练习 3：坐标位置判断

考点：元组、解包、函数、循环。

要求写函数：

```python
def classify_point(point: tuple[int, int]) -> str:
    ...
```

分类规则：

- `(0, 0)`：原点
- `(0, y)`：Y轴
- `(x, 0)`：X轴
- `(x, y)`：普通坐标
- 其他结构：无效坐标

测试数据：

```python
points = [(0, 0), (2, 0), (0, 5), (3, 4)]
```

## 练习 4：学生字典匹配

考点：字典、条件判断、守卫条件。

输入示例：

```python
student = {"name": "张三", "score": 85}
```

要求：

- 匹配出 `name` 和 `score`。
- 根据分数输出优秀、及格、不及格。
- 缺少字段时输出“信息不完整”。

优先级：中低。普通 `if / elif` 成绩判断必须先会。

## 练习 5：列表命令解析

考点：字符串 `split()`、列表、结构解包、类型转换。

输入被拆成：

```python
command = ["add", "张三", "90"]
```

匹配规则：

- `["add", name, score]`：添加学生
- `["delete", name]`：删除学生
- `["list"]`：显示学生
- `["quit"]`：退出
- 其他：无效命令

## 练习 6：文件命令处理

考点：文件、字符串、列表、异常。

文件内容示例：

```text
add,张三,85
add,李四,92
delete,王五
list
```

要求：

1. `with open()` 逐行读取。
2. `strip()` 去除换行。
3. `split(",")` 转成列表。
4. 用 `match` 判断命令结构。
5. 分数转换失败时捕获 `ValueError`。
6. 结构错误时记录“无效命令”。

优先级：低。只有 Day 5 文件异常通过后才做。
