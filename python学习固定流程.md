# Python 学习固定流程

每次学习一个 Day 的语法点，严格按下面顺序走，不要跳步。

---

## 准备阶段（CC做，1-2分钟）

- [ ] CC生成 `theory_xxx.md`（理论文档，10分钟内能读完）
- [ ] CC生成 `practice_xxx.py`（带TODO标注的练习文件）
- [ ] 确认文件放在 `C:\Users\50469\python-learning\` 下

## 执行阶段（你做，20-30分钟）

### 第一步：读理论（5分钟）
- [ ] 打开 theory 文件，读完全部内容
- [ ] 有不懂的标记出来，读完问CC
- [ ] 每个代码示例在脑子里跑一遍

### 第二步：手敲练习（15-20分钟）
- [ ] 打开 practice 文件
- [ ] **每个TODO自己敲代码，不复制粘贴**
- [ ] 命令不会就问，但代码必须手打
- [ ] 不会做先试，试错了也要试

### 第三步：破坏实验（必做）
- [ ] 至少做1个破坏实验（故意写错 → 看报错 → 理解报错）
- [ ] 把报错信息和你发现的规律写成注释

### 第四步：跑通验证
- [ ] `python practice_xxx.py` 跑通，没有报错
- [ ] 输出和预期一致

## 收尾阶段（CC做，5分钟）

### 第五步：记录错误
- [ ] CC把今天所有错误写入 `progress/日期_progress.md`
- [ ] 格式：现象 → 我的猜测 → 真实原因 → 下次怎么定位
- [ ] **最后一列最重要** —— 迭代你的排查策略

### 第六步：口头复述（2分钟）
- [ ] 不看代码，说出今天学了哪几个语法点
- [ ] 说出今天遇到的一个错误及其原因
- [ ] 说不出 → 回去看一眼文件 → 再试

### 第七步：安排回顾
- [ ] CC在24小时后提醒回顾

---

## 标准流程速记卡

```
读 → 敲 → 破 → 跑 → 记 → 述 → 回顾
 ↑    ↑    ↑    ↑    ↑    ↑     ↑
theory TODO 故意 python 错误 口头   CC
10min 写  写错 验证  记录  复述   提醒
```

## 三个永不

1. **永不跳过破坏实验** —— 写错比写对学得更深
2. **永不复制粘贴** —— 每个字符从手指打出来
3. **永不跳过错误记录** —— 没记=白踩坑
