## 你真正需要补哪些

### 必补 1：`raise` 主动抛异常

老师代码：

```python
if len(password) <= 6:
    raise Exception("密码长度小于6")
```

后端项目里对应：

```python
if company is None:
    raise CompanyNotFoundError(company_id)
```

你要理解：

```text
raise 不是程序出错才有
raise 是你主动告诉程序：这里发生了一个不允许继续的情况
```

在后端里，`raise` 常用于：

```text
资源不存在
参数冲突
状态不允许变更
重复创建
权限不足
```

---

### 必补 2：自定义异常

老师代码：

```python
class ShortInputError(Exception):
    def __init__(self, length, atleast):
        self.length = length
        self.atleast = atleast
```

后端项目里应该迁移成：

```python
class CompanyNotFoundError(Exception):
    ...

class DuplicateCompanyNameError(Exception):
    ...
```

意义：

```text
不要所有业务错误都用 Exception
不同错误应该有不同异常类
这样 route 层才能转成不同 HTTP 状态码
```

例如：

```text
CompanyNotFoundError       → 404
DuplicateCompanyNameError  → 409
InvalidStatusTransition    → 400
```

---

### 必补 3：异常传递

老师代码链路：

```text
show_area()
→ calc_area()
→ get_width()
→ ZeroDivisionError
→ 一层层向上传
→ show_area 捕获
```

FastAPI 项目链路：

```text
main.py route
→ service.get_company()
→ repository.get_company()
→ service 发现不存在
→ raise CompanyNotFoundError
→ route 捕获
→ raise HTTPException(404)
```

你要吃透这个，因为它解释了：

```text
为什么 service.py 可以 raise 错误
为什么 main.py 能捕获这个错误
为什么 repository 不应该直接 raise HTTPException
```

---

### 必补 4：`except Exception as error`

老师代码：

```python
except Exception as error:
    print("出错了，原因：", error)
```

你要知道两点：

1. `error` 是异常对象，可以打印错误原因。
2. 项目里不要随便捕获所有异常，否则会吞掉真正 bug。

推荐规则：

```text
学习阶段：可以用 except Exception as error 看报错
项目代码：优先捕获明确异常
```

## 了解即可的内容

### `assert`

老师讲：

```python
assert num_two != 0, "除数不能为0"
```

你现在知道它会在条件为 False 时抛 `AssertionError` 就行。

注意：

```text
assert 适合调试和内部不变量检查
不适合处理用户输入
```

用户输入错误应该用：

```text
Pydantic 校验
raise 业务异常
HTTPException
```

### 裸 `except`

老师有：

```python
except:
    print("出错了")
```

项目里不推荐这样写。

原因：

```text
它会捕获太多东西
容易把真正的 bug 藏起来
也不利于定位问题
```

推荐：

```python
except ValueError as error:
    ...

except CompanyNotFoundError as error:
    ...
```

## 和 FastAPI 的直接关系

### 1. Pydantic 负责 422

```text
请求体字段类型错
字段缺失
字段不符合 Field 约束
```

这些通常不需要你手写 `try/except`，FastAPI 会自动返回 422。

### 2. service 负责业务异常

例如：

```text
公司不存在
岗位不存在
重复投递
状态流转非法
```

这些应该在 service 层：

```python
raise XxxError(...)
```

### 3. route 负责转 HTTP

```python
try:
    return service.get_company(company_id)
except CompanyNotFoundError as exc:
    raise HTTPException(status_code=404, detail=str(exc)) from exc
```

这就是老师“异常传递”和“重新引发异常”在 FastAPI 里的实际应用。
