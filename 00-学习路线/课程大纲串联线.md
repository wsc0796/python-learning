## 逐章串联：每章的"为什么在这"和"通向哪里"

### 第一章 Python 基础 → 第二章 面向对象

**连接逻辑**：

```
变量 → 数据类型 → 运算符 → 条件循环 → 字符串 → 容器 → 函数 → 文件 → 异常 → 模块
                                                                              │
                                                                   ┌──────────┘
                                                                   ▼
                                              第二章：把这些组织成 类 和 对象
```

每一个基础语法点通向的进阶位置：

| 基础知识点 | 在第二章被用到 | 在后续章被用到 |
|-----------|-------------|-------------|
| 变量、数据类型 | `self.name = name` | Pydantic Field 类型声明 |
| 条件语句 + 循环 | `__init__` 里的判断、遍历 | CRUD 的筛选/遍历 |
| 字符串操作 | `__str__`、fstring | LLM prompt 模板 |
| 列表/字典/元组/集合 | 存储对象集合 | JSON 数据、向量检索结果 |
| 函数 | 方法 = 类里的函数 | `@app.get()` 路由函数 |
| 文件操作 | `json.load/dump` | Milvus 数据持久化 |
| 异常处理 | `raise ValueError` | `HTTPException(404)` |
| 模块 import | `from models import Task` | LangChain 各组件导入 |

> **本章在本项目的对应模块**：`01-fstring` ~ `27-regex` 中的 25 个基础模块

### 第二章 面向对象进阶 → 第三章 学生管理系统

**连接逻辑**：

```
类定义 → self → __init__ → 继承 → 封装 → 多态 → 类方法 → 闭包 → 装饰器 → 生成器
                                                                              │
                                                              ┌───────────────┘
                                                              ▼
                                         第三章：用面向对象思想做一个完整项目
```

这门课在面向对象之后立刻安排"学生管理系统"项目是有意的：

- **继承** → 学生/教师都继承自 `Person` 基类
- **封装** → 成绩数据私有化，通过方法访问
- **多态** → 不同角色重写 `display_info()` 方法
- **文件操作** → 学生数据存入 JSON/CSV
- **异常处理** → 输入校验、数据不存在时的处理

第三章是第一章+第二章的"综合体检"——如果你的面向对象没学透，写学生管理系统时一定会卡。

> **本章在本项目的对应模块**：`14-advanced-oop`、`15-oop-deep-dive`、`11-closure-decorator`、`12-design-patterns`、`13-multithreading`、`17-context-iterator-async`、`22-copy-os`

### 第三章 学生管理系统 → 第四章 MySQL

**连接逻辑**：

```
第三章：用 Python list/dict 管理 100 个学生
                    │
                    ▼ 问题来了：数据关掉程序就没了 → 需要持久化
                    │
第四章：用 MySQL 存数据，SQL 查询代替 Python 循环遍历
```

这是一个**隐含的教学设计**——先让你用 Python 手写增删改查，感受痛点，再引入数据库：

| 第三章的手写方式 | 第四章的 SQL 方式 | 为什么要升级 |
|---------------|-----------------|------------|
| `[s for s in students if s.id == 5]` | `SELECT * FROM students WHERE id=5` | 遍历 O(n)，索引 O(1) |
| `students.append(new_student)` | `INSERT INTO students VALUES(...)` | 内存易失，数据库持久 |
| `[s for s in students if s.name like '%张%']` | `WHERE name LIKE '%张%'` | Python 手写慢，SQL 引擎优化 |
| `max(s.score for s in students)` | `SELECT MAX(score) FROM students` | 聚合函数比你手写快 |

窗口函数 (row_number/rank/dense_rank) 是这一章的最高点——它直接通到第五章的 Pandas 分组聚合，也通到 RFM 项目的用户分层。

> **本章在本项目的对应位置**：`28-sky-takeout-internalization`（苍穹外卖用了 MySQL）、`黑马点评-速通计划.md`（Redis + MySQL）

### 第四章 MySQL → 第五章 数据处理与统计分析

**连接逻辑**：

```
第四章：SQL 查出来的数据是表格
                    │
                    ▼ 问题来了：表格怎么变成图表？怎么发现规律？
                    │
第五章：用 Python 做 SQL 做不到的事 —— 矩阵运算、统计推断、可视化
```

这一章从"管数据"跨越到"用数据"：

| 第四章（数据库思维） | 第五章（分析思维） | 能力升级 |
|-------------------|-----------------|---------|
| `SELECT AVG(score) FROM ...` | `df['score'].mean()` | 从单值统计到 DataFrame 整体操作 |
| `GROUP BY department` | `df.groupby('department').agg(...)` | 从分组查询到分组+多函数聚合 |
| (无) | `df.corr()` 相关矩阵 | 从查数据到发现变量间关系 |
| (无) | `plt.plot(df['date'], df['sales'])` | 从看表格到看趋势 |

**Jupyter Notebook 在这一章被引入不是偶然的**——它的"代码→结果→图表"即时反馈循环，正好是数据分析需要的探索式编程，也恰好是后面 LLM 实验的标准环境。

RFM 项目是这一章的"总验收"：
```
读取 CSV (Pandas) → 数据清洗 (缺失值处理) → 计算 R/F/M 三个维度 (NumPy/apply)
→ 用户分组 (groupby/qcut) → 可视化 (Matplotlib) → 导出 Excel/MySQL
```

> **本章在本项目的对应模块**：`29-data-analysis-intro`、`29-numpy`、`30-pandas`、`31-matplotlib`、`32-data-analysis-rfm`

### 第五章 数据处理 → 第六章 扣子平台 + 第七章 LLM 基础

**连接逻辑**：

```
第五章：你学会了用 Python 处理数据、做统计、画图表
                    │
                    ▼ 现在换个角度：让 AI 模型帮你理解数据、生成内容
                    │
第六章：先在扣子平台"感受" LLM 能做什么（不写代码，先建立直觉）
第七章：再拆开看 LLM 内部怎么工作（走进黑箱）
```

这是一个**先感性后理性**的教学设计：

| 阶段 | 你在做什么 | 对应的认知层次 |
|------|----------|-------------|
| 第六章 扣子实操 | 拖拽节点搭工作流，LLM 是一个黑箱节点 | "LLM 能干什么" |
| 第七章 LLM 原理 | BLEU/ROUGE/PPL 指标、Encoder-Decoder 架构 | "LLM 为什么能做到" |

**关键连接点**：

- 第六章的"代码块节点" → 你的 **Python 基础**（第一章）在这里直接上场
- 第六章的"JSON 数据结构" → 你的 **字典** 知识（第一章）在这里被重新激活
- 第六章的"变量体系/数据传递" → 就是编程里的 **参数传递**（函数那章）
- 第七章的"BLEU/ROUGE 指标" → 就是第五章学的 **统计计算**（概率、n-gram 计数）
- 第七章的"Decoder-Only 架构" → 需要理解 **循环/迭代/自回归**（第一章的 for/while）

> **本章在本项目的对应模块**：`18-llm-coze-workflow`（第六章）、`19-llm-finetune-rag-agent-projects`（第七~九章）

### 第七章 LLM 基础 → 第八章 微调与提示词工程

**连接逻辑**：

```
第七章：GPT-1/2/3 是怎么训练的 → 模型有了"通用能力"
                    │
                    ▼ 问题来了：通用模型不懂你的业务场景怎么办？
                    │
第八章：两种路线 —— Prompt 工程（不训练模型，改提示词）
                         Fine-Tuning（微调模型参数，适配你的数据）
```

这是一个**成本决策框架**：

```
Prompt-Tuning (Soft-Prompt / P-Tuning)
  │  成本低，不需要改模型权重，只学一小段"提示向量"
  │
  ├─→ In-Context Learning + CoT（连向量都不学，直接在 prompt 里给例子+思维链）
  │
  └─→ Fine-Tuning（LoRA / Adapter / PEFT）
      成本高，但效果更深，适合垂直领域
```

**关键连接点**：

- In-Context Learning（情景学习）→ 就是 **Few-Shot Prompt**，你在第六章扣子里已经用过（给 LLM 节点写示例）
- CoT（思维链）→ 就是让 LLM **分步推理**，类似你写代码时的逐步调试思路
- LoRA → 不修改原模型，只加一小层可训练的矩阵 → 类似 **装饰器模式**（不改原函数，包装一层新行为）

> **本章在本项目的对应模块**：`19-llm-finetune-rag-agent-projects`

### 第八章 微调 → 第九章 大模型开发与应用

**连接逻辑**：

```
第八章：模型"懂你的行业了"（微调完）
                    │
                    ▼ 问题来了：怎么让模型"做事"而不是"聊天"？
                    │
第九章：Function Call → Agent → RAG → LangChain
```

这一章的三个核心技术是逐层递进的：

```
Function Call（模型能调你的函数）
  │  模型输出不是自然语言，而是 {"function": "search_database", "parameters": {...}}
  │
  ├─→ Agent（模型能自己决定"下一步做什么"）
  │    模型拿到 Function Call 的结果后，自己判断是继续调函数还是直接回答用户
  │    核心范式：ReAct = Reason（推理）+ Act（行动）
  │
  └─→ RAG（检索增强生成）
       模型回答之前，先去向量数据库里找相关文档
       = 给模型配了一个"外部记忆"
```

**这里的每一个技术都需要前面的地基**：

| 技术 | 需要的前置知识 | 来自第几章 |
|------|-------------|----------|
| Function Call 的 JSON 参数 | 字典、JSON 结构 | 第一章（dict）、第六章（JSON） |
| Agent 的决策循环 | while 循环、条件判断 | 第一章 |
| RAG 的向量检索 | 矩阵运算、相似度计算 | 第五章（NumPy） |
| Milvus 增删改查 | SQL 的 CRUD 思维 | 第四章（MySQL） |
| LangChain 的 Chain | 函数组合、管道模式 | 第一章（函数）、第二章（设计模式） |

> **本章在本项目的对应模块**：`19-llm-finetune-rag-agent-projects`、`20-agent-architecture-harness`

## 全课程唯一主线：数据的一生

如果你只能记住一条线，记这条：

```
数据从哪来 ──→ 数据怎么存 ──→ 数据怎么查 ──→ 数据怎么分析 ──→ 数据喂给 AI
    │               │               │               │                │
  第一章           第四章           第四章           第五章          第六~十章
  input()         INSERT         SELECT          Pandas          LLM/RAG
  文件读取         MySQL          窗口函数         Matplotlib      Agent
```

**同一个"数据"穿过十章的完整旅程**：

```text
用户在网页输入一条评论 (第一章：input → 字符串)
  → 存到 MySQL (第四章：INSERT INTO comments)
    → 查出来做分析 (第四章：SELECT ... GROUP BY sentiment)
      → 画情感分布图 (第五章：df.plot(kind='bar'))
        → 微调 ChatGLM 做自动分类 (第七~八章)
          → Agent 自动回复差评 (第九章：Function Call 调客服API)
```

这就是这门口课设计成十章的内在逻辑——**不是在学十个独立的东西，而是在学一根链条上的十个环节**。
