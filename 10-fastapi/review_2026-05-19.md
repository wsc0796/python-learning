## 今日遇到的 13 个困难

### 1. 类名+括号 = 造对象
**代码：** `task_repository = JsonTaskRepository(DATA_FILE)`

**当时不懂：** 括号里放 DATA_FILE 是什么意思

**核心理解：** `ClassName(参数)` 括号传给 `__init__`，用来初始化对象属性。类比 `Dog("大黄")`，class 是模具，括号是压实物。

---

### 2. dependencies.py 存在的意义
**当时不懂：** 这句话看了也就看了，不知道有什么用

**核心理解：** "创建对象"和"使用对象"分家。dependencies.py 是唯一的组装车间——造对象 + 连线 + 提供取货口。好处：换存储只改一处，测试时偷梁换柱。

---

### 3. Annotated + Depends() 语法
**代码：** `TaskServiceDep = Annotated[TaskService, Depends(get_task_service)]`

**当时不懂：** 这一段完全看不懂

**核心理解：**
```
Annotated[ TaskService , Depends(get_task_service) ]
        ~~~~~~~~~~~   ~~~~~~~~~~~~~~~~~~~~~~~~
          类型标签           取货指令
```
`Depends(函数)` = 告诉 FastAPI "这个参数你别等用户传，帮我去执行那个函数拿值"。`Annotated` 本身不干活，只是贴标签的容器。

---

### 4. model_dump() 对象 → 字典
**代码：** `task.model_dump()`

**当时不懂：** 笔记里没有这个内容

**核心理解：** Pydantic 对象自带的方法，把对象转成普通字典。必须这样转是因为 `json.dump()` 不认识 Pydantic 对象，只认识 dict。

---

### 5. model_validate() 字典 → 对象
**代码：** `TaskRead.model_validate(raw_task)`

**核心理解：** `model_dump()` 的逆向操作——从字典创建 Pydantic 对象，同时校验。比 `**data` 解包更安全（后者不校验脏字段）。

---

### 6. model_copy() 创建副本
**代码：** `task.model_copy(update={"completed": True})`

**核心理解：** 不修改原对象，新建一个对象并改掉指定字段。等于手写 `TaskRead(id=..., title=..., completed=True)` 但省代码。

---

### 7. max(..., default=0) 空列表兜底
**代码：** `next_id = max((task.id for task in tasks), default=0) + 1`

**核心理解：** 列表有内容取最大 → +1；列表空时兜底返回0 → +1 = 1（第一id从1开始）。`default=0` 防止 `max()` 在空列表时抛 `ValueError`。

---

### 8. Protocol 行为约定
**代码：** `class TaskRepository(Protocol):`

**当时不懂：** 这 Protocol 存在的作用是什么

**核心理解：** Python 的 structural subtyping——不看你声明了什么，只看你有什么方法。`JsonTaskRepository` 不需要写继承关系，只要有 `list_tasks()` 和 `save_tasks()` 就自动满足。类比 Go 的 interface。

---

### 9. Pydantic 校验输入 vs 规范输出的完整流程
**当时不懂：** Pydantic 如何校验输入数据和规范输出数据

**核心理解：** 一次 POST 请求的完整旅程——
```
用户JSON → FastAPI(TaskCreate校验输入) → Service(加上id/completed) 
→ Repository(model_dump转字典) → JSON文件
→ FastAPI(TaskRead规范输出) → 用户收到干净JSON
```
- TaskCreate = 入库关（"脏数据别进来"）
- TaskRead = 出库关（"输出格式统一"）

---

### 10. save_tasks() 逐行看不懂
**核心理解：**
```python
self.filename.parent.mkdir(...)      # 确保目录存在
payload = {"tasks": [model_dump()]}  # 对象列表 → 字典列表
with self.filename.open("w") as f:   # 打开文件
    json.dump(payload, f, ...)       # 写入JSON
```

---

### 11. bool | None = None 看不懂
**代码：** `def list_tasks(self, completed: bool | None = None) -> list[TaskRead]:`

**核心理解：**
- `bool | None` = 参数可以是 True、False、None 三种值
- `= None` = 不传则默认为 None
- `None` 是哨兵值：当 True/False 已有语义（筛选已完成/未完成）时，用 None 代表"不过滤"

---

### 12. task.completed == completed 两个同名变量分不清
**代码：** `if task.completed == completed`

**核心理解：**
```python
task.completed   ← 每个任务自己的字段值（遍历出来的）
completed        ← 调用者传进来的参数值（True/False/None）
```
判断逻辑：
- `completed is None` → 全返回
- `completed is True` → 只返回已完成
- `completed is False` → 只返回未完成

---

### 13. json.load() 和 json.dump() 方向记反
**纠正口诀：**
```
dump  = 倒出去（序列化）：字典 → JSON字符串 → 写入文件
load  = 装进来（反序列化）：文件 → JSON字符串 → Python字典

镜像对称：
  save_tasks: 对象 ─→ model_dump() ─→ dict ─→ json.dump() ─→ 文件
  list_tasks:  文件 ─→ json.load()  ─→ dict ─→ model_validate() ─→ 对象
```

## 今日掌握的口诀清单

| 口诀 | 含义 |
|------|------|
| 括号 = 造对象 | `ClassName(args)` = 传参数给 `__init__` |
| 组装车间 | dependencies.py 只负责创建和连线 |
| 对象转字典 | `model_dump()` |
| 字典转对象 | `model_validate()`（带校验）|
| 副本改字段 | `model_copy(update={...})` |
| 空列表兜底 | `max(seq, default=0)` |
| Protocol = 拍胸脯 | 只要你有这些方法我就信任你 |
| `bool \| None` | 参数可以是 True/False/None |
| `None` 做哨兵值 | True/False 已有语义时用 None 代表"未指定" |
| dump=倒，load=装 | dump 写出去，load 读进来 |
| 参数名 vs 字段名 | `task.xxx == xxx`：前面字段，后面参数 |
| 需求→步骤→代码 | 不要直接写代码，先翻译成人话步骤再找对应语法 |
| 照着抄只改名字 | 左边开原文件右边写新文件，逻辑照抄，只改 Task→Note |
